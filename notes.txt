# PortDesk --- Notes & Documentation

## Overview
This document covers essential security mechanisms (PIN protection, whitelist/blacklist), known issues and their workarounds, platform-specific behavior (Windows, macOS, Linux), and screen streaming performance characteristics.

---

## Quick Fix (Priority 1)
* Keyboard not responding? Try running the server from the dedicated launch file (start_portdesk.bat) or running it minimized/hidden.
* On Windows: Open start_portdesk.bat (configured for start /min python "portdesk-server.py"). This resolves focus/input issues in most cases.
* On Linux/Mac: Use start_portdesk_linux.sh or start_portdesk_mac.sh to run the server in the background.

For any issue not covered here, comment it in discussions

---

## Security & Access Control

### 1. Forgotten PIN / Pattern
If you are locked out:
Open portdesk_security.json and delete the "pins" entry for your IP address and save the file. then The server reloads automatically within 2 seconds.

### 2. PIN Brute Force Protection
* Attempts: 5 failures trigger a lockout.
* 1st Lockout: 60 seconds.
* 2nd Lockout: 180 seconds.
* 3rd+ Lockout: 300 seconds (fixed).
* Successful login resets the counter for that IP.

### 3. Whitelist & Auto-Blacklist
* Whitelist: New devices prompt: "Add to whitelist? (y/n)".
* Auto-Blacklist: If a device is rejected 3 times, it is permanently blocked.
* Reset: Owner can remove IPs via terminal or local security JSON.

### 4. TOFU Certificate Warning (False Alarm)
If you regenerate cert.pem / key.pem, the client will show a Man-in-the-Middle (MITM) warning. This is a False Alarm due to the fingerprint change.
* Fix: Go to Settings tab-> reset trusted cert.

## Command Line & Runtime

### CLI Launch Flags
Example: python portdesk-server.py --help

| Flag | Default | Description |
| :--- | :--- | :--- |
| -p / --port | 5000 | Port to listen on |
| -s / --ssl | off | Enable HTTPS |
| --watch-only | off | Disable all input (View only) |
| --backend dxcam|mss | Force screen capture backend |
| --scale FACTOR | 1.0 | Downscale stream (e.g. 0.75) |
| --grey | off | Greyscale mode (low bandwidth) |
| --upload-limit | unlimited | Max file size (e.g. 1GB) |

### Terminal Commands (While Running)
| Command | Description |
| :--- | :--- |
| y / n | Approve or Reject last request |
| unblock <ip> | Remove IP from blacklist |
| kick all | Disconnect everyone |
| lockdown off | Lift active lockdown |
| restore security [1-3] | Restore from backup (.bak1-.bak3) |

---

## Performance & Streaming (Latency)

* H264 Mode: Requires FFmpeg. Lowest bandwidth, best quality.
* JPEG Mode: Fallback. Higher bandwidth, no FFmpeg needed.
* Latency Factors: Server CPU (encoding), WiFi strength (transfer), and Browser engine (decoding).

---

## Known Issues
* Python 3.14: Asyncio issues on Windows. Use 3.11 or 3.12.
* H264: Requires FFmpeg in PATH.
* Gyroscope/Mic: Requires HTTPS (run gen_cert.py).
* Multi-monitor: dxcam locks to one monitor at startup.

# PortDesk --- ملاحظات وتوثيق

## إصلاح سريع (الأولوية الأولى)
* الكيبورد لا يستجيب؟ جرب تشغيل السيرفر من ملف start_portdesk.bat أو تشغيله بشكل minimized. في ويندوز، هذا يحل أغلب مشاكل الـ focus.
* في لينكس وماك: استخدم سكربتات .sh لتشغيل السيرفر في الخلفية.

---

## نظام الأمان والحماية

### 1. حماية الـ PIN من التخمين
* عندك 5 محاولات قبل القفل.
* مدد القفل: دقيقة واحدة ثم 3 دقائق ثم 5 دقائق (ثابتة بعد ذلك).
* أثناء القفل، تتعطل الأزرار تماما لمنع أي تحايل.

### 2. القائمة البيضاء والسوداء
* الموافقة: عند اتصال جهاز جديد، يجب أن تضغط y في التيرمينال للموافقة.
* الحظر التلقائي: الرفض 3 مرات يضع الجهاز في القائمة السوداء نهائيا.
* إدارة المستخدم: الكلايانت يمكنه إزالة نفسه فقط من الإعدادات.

### 3. تحذير الشهادة (TOFU)
* عند تغيير ملفات الـ SSL، سيظهر تحذير MITM. هذا إنذار كاذب بسبب تغير بصمة الشهادة.
* الحل: من إعدادات المتصفح داخل التطبيق اختر Reset Trusted Cert.

---

## بث الشاشة والأداء
* الجودة: تختلف حسب كثافة بكسلات شاشة الكلايانت (DPR) وقوة معالج السيرفر.
* وضع H264: الأفضل والأسرع، يتطلب وجود FFmpeg في النظام.
* التأخير (Latency): عبارة عن سلسلة تبدأ من (قوة المعالج في الضغط) ثم (سرعة الـ WiFi) ثم (سرعة المتصفح في العرض).

---

## مشاكل معروفة
* Python 3.14: يسبب مشاكل في الـ Asyncio على ويندوز. ينصح بـ 3.11 أو 3.12.
* الجيروسكوب والميكروفون: لا يعملان إلا عبر HTTPS.
* dxcam: يحتاج إعادة تشغيل السيرفر عند تغيير الشاشة في إعدادات الشاشات المتعددة.
